# OctoAcme Role Interaction Matrix

This matrix shows typical interactions between roles across the project lifecycle and provides a simple checklist for responsibilities at each phase.

## Phases
- Initiation
- Planning
- Execution
- Release
- Close

## Interaction Matrix

| Role \ Phase | Initiation | Planning | Execution | Release | Close |
|---|---:|---:|---:|---:|---:|
| Product Manager | Lead: define objectives, success metrics, stakeholders | Lead: prioritize backlog, define acceptance criteria | Collaborate: clarify scope, accept features | Communicate: release messaging | Review outcomes, measure success |
| Project Manager | Coordinate: timeline & resources | Coordinate: dependencies, schedule | Track: progress, escalate risks | Coordinate: deploy windows & comms | Capture lessons, close activities |
| Scrum Master | Consult: advise on team readiness | Facilitate: sprint planning, team capacity | Facilitate: daily standups, remove blockers | Support: release readiness | Facilitate: retrospectives |
| Business Analyst | Consult: requirements intake | Define: detailed requirements & flows | Support: clarify edge cases, UAT scripts | Validate: migration/data concerns | Validate: acceptance & closure |
| UX/UI Designer | Consult: user research inputs | Design: wireframes, prototypes | Collaborate: handoff & reviews | Validate: UI checks & smoke tests | Archive designs, gather feedback |
| Developers | Consult: technical feasibility | Estimate: stories & tasks | Build: implement, test | Prepare: release artifacts & fixes | Support: post-release fixes |
| QA/Testing | Consult: test strategy | Define: test plans & criteria | Test: integration & regression | Execute: smoke & acceptance tests | Report: verification & lessons |
| Release Manager | N/A | Plan: release runbook & checklist | Coordinate: deployment steps | Lead: deployment & rollback | Report: post-release health |
| Stakeholders | Provide direction | Review: priorities | Feedback: beta/UAT | Communicate: adoption | Sign-off: acceptance |

## Role Responsibility Checklist (copy into project README or sprint template)

- Initiation
  - Product Manager: Draft one-pager and success metrics
  - Project Manager: Identify initial timeline and stakeholders
  - Business Analyst: Capture initial requirements and assumptions
  - UX/UI Designer: Gather user research inputs if available

- Planning
  - Product Manager: Prioritize backlog and define acceptance criteria
  - Project Manager: Create release plan and highlight dependencies
  - Scrum Master: Facilitate planning and ensure DoD
  - Business Analyst: Finalize detailed requirements
  - UX/UI Designer: Provide wireframes and interaction specs
  - QA: Draft test plan and acceptance tests

- Execution
  - Developers: Implement stories and write tests
  - QA: Run integration and regression tests
  - Scrum Master: Remove blockers and maintain team flow
  - Business Analyst: Clarify requirements and support UAT

- Release
  - Release Manager: Run deployment checklist and communications
  - QA: Execute smoke tests and sign-off
  - Product Manager: Coordinate release messaging
  - Project Manager: Confirm stakeholder notifications

- Close
  - Project Manager: Document lessons learned and close tasks
  - Product Manager: Measure outcomes vs success metrics
  - Scrum Master: Facilitate retrospective and action items
  - Release Manager: Archive release artifacts
